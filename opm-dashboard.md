# OPM Data Dashboard

## Goal
Public-facing, visually minimalist R Shiny dashboard for exploring federal workforce changes over time. Key focuses: scientists/STEM workers, geographic concentration over time, agency and sub-agency breakdowns. Part of a work portfolio. Future phase: embedded AI assistant.

## Source Data
- BigQuery table: `mapping-patents:opm_data.employment` (~420M rows, all-STRING columns)
- Data dictionary: `~/Downloads/opm-data/FWD Data Dictionary.xlsx`
- Key columns in actual data: `file_year`, `file_month`, `file_type`, `agency`, `agency_subelement`, `duty_station_state_abbreviation`, `duty_station_country_code`, `stem_occupation_type`, `occupational_group`, `occupational_series`, `annualized_adjusted_basic_pay`, `work_schedule`
- NOTE: `occupational_category` and `nsftp_indicator` are in the data dictionary but NOT in the actual files

## BigQuery Summary Tables

### `opm_data.agency_monthly`
Groups: file_year, file_month, agency, agency_subelement, occupational_group, work_schedule → headcount, avg_pay

### `opm_data.stem_monthly`
Groups: file_year, file_month, agency, agency_subelement, stem_occupation_type, duty_station_state_abbreviation → headcount
- Excludes: `stem_occupation_type IN ('ALL OTHER OCCUPATIONS', 'UNSPECIFIED', '')`
- Valid stem_occupation_type values: SCIENCE, TECHNOLOGY, ENGINEERING, MATHEMATICS, HEALTH OCCUPATIONS

### `opm_data.geo_monthly`
Groups: file_year, file_month, duty_station_state_abbreviation, agency, stem_occupation_type → headcount
- US only (duty_station_country_code = 'US'), excludes NULL/empty states

SQL scripts: `sql/01_create_agency_monthly.sql`, `02_create_stem_monthly.sql`, `03_create_geo_monthly.sql`
**Re-run all 3 after each new month of employment data is uploaded.**

### `opm_data.agency_analyses`
Pre-computed statistical analyses for every agency and sub-agency with >= 1,000 employees.
- **Generated by**: `scripts/generate_analyses.R` (standalone batch script — NOT in `R/` dir, Shiny auto-sources everything in `R/`)
- **Schema**: entity_type, agency, agency_subelement, analysis_period_start, analysis_period_end, analysis_generated_at, headcount_start, headcount_end, pct_change_5yr, overall_slope, overall_slope_pvalue, narrative_text, stats_json
- **Stats method**: linear regression (headcount ~ time_index) over last 60 months; Z-test on slope difference for occupational group divergence from entity overall
- **Handles NA/NaN**: near-zero SE from stable headcounts flagged as "not estimable" in narrative
- **Key fix**: BigQuery returns INT64 as `integer64` (bit64 package) — convert with `as.numeric()` at load time or `bind_rows()` fails with vctrs type error
- **Schema reference**: `sql/04_create_agency_analyses.sql`
- **Re-run**: `source("scripts/generate_analyses.R")` after each monthly data/SQL rebuild

### Known data gap
- Feb 2025 was missing from OPM download initially; manually downloaded and uploaded, tables rebuilt
- OPM published quarterly snapshots (Mar/Jun/Sep/Dec only) from 2005–2011; monthly from ~2012 onward — early "missing" months are expected

`shiny`, `bslib`, `bigrquery`, `DBI`, `dplyr`, `plotly`, `leaflet`, `sf`, `tigris`, `scales`, `reactable`, `broom`, `jsonlite`, `glue`, `renv`

```

├── app.R                    # Main entry: data load, UI, server
├── R/
│   ├── bq_queries.R         # BQ auth + query functions (all runtime helpers go here)
│   ├── plots.R              # All plotly + reactable chart functions

├── scripts/
│   └── generate_analyses.R  # Batch analysis script (NOT in R/ — Shiny auto-sources R/)
├── sql/                     # BigQuery CREATE TABLE scripts
├── www/custom.css           # Brand styles
└── .gitignore               # Excludes .Renviron, *.json (credentials)
```

### Data loading strategy
- All summary tables loaded at startup into memory (fast, cheap)
- Pre-aggregated further at startup into working frames (agency_by_date, total_headcount_ts, stem_by_date)
- `bindCache()` on all reactive outputs for instant re-renders
- BigQuery connection closed immediately after startup load (`dbDisconnect(con)`)

### Security
- Service account JSON written to tempfile for auth, cleaned up via `on.exit(unlink(tmp))` in `bq_connect()`
- No user input ever touches a database query — all filtering is in-memory
- No credentials in repo; `.gitignore` excludes `.Renviron` and `*.json`
- GCP project ID (`mapping-patents`) visible in code — not a secret; IAM protects actual access

## Dashboard Tabs

### Tab 1 — Overview
- 4 KPI value_boxes: total headcount, YoY change %, STEM share %, STEM headcount YoY change %
- Total federal headcount time series (plotly)
- YoY cards show "N/A" gracefully if prior year month is missing

### Tab 2 — By Agency
- Sidebar: multi-select agency picker (selectizeInput, default: HHS), sub-agency filter, toggle "Show each agency separately"
- 7/5 column split: trend line (left) + reactable ranking table (right)
- Combined mode (default): single teal line summing all selected agencies
- Split mode: one line per agency with legend
- Reactable table: sub-agencies ranked by headcount with inline teal bars

### Tab 3 — Scientists & STEM
- Sidebar: STEM category checkboxes using short labels via choiceNames/choiceValues (all selected by default), agency filter
- 7/5 column split: stacked area chart (left) + reactable ranking table (right)
- Stacked area: STEM types over time with brand color palette, short legend labels

### Tab 4 — Geography
- Sidebar: year slider (dynamic max from `max(geo_data$file_year)`), metric radio (Headcount / YoY Change count / YoY Change %), STEM-only checkbox, agency filter
- Leaflet choropleth: all 50 states + DC via `tigris` (Census boundaries, cached)
- Default view: centered on lower 48 (lng=-96, lat=38, zoom=4)
- Headcount metric: sequential teal scale
- YoY metrics: symmetric diverging scale (red=negative, white=zero, teal=positive)
- `sf_use_s2(FALSE)` required — tigris/Census data has minor edge issues S2 rejects

## Known Issues / Decisions
- `shinycssloaders` spinner breaks dynamic plot resizing — removed
- `occupational_category` is in data dictionary but absent from actual files; use `occupational_group`
- `maps::map("state")` only covers contiguous 48; switched to `tigris::states()` for AK/HI
- Shiny auto-sources ALL `.R` files in the `R/` directory — batch scripts must go in `scripts/` instead

## Monthly Maintenance Checklist
After each new OPM data upload:
1. Re-run `sql/01_create_agency_monthly.sql` in BigQuery console
2. Re-run `sql/02_create_stem_monthly.sql`
3. Re-run `sql/03_create_geo_monthly.sql`
4. Run `source("scripts/generate_analyses.R")` in R
5. Run `rsconnect::deployApp(appDir = ".", appName = "fed-govt-employment-dashboard")` to redeploy
